3. Implementing Logistic Regression¶

Recall that our cost function is as follows:

$$J(w)=-\frac{1}{m} \sum_{i=1}^{m} \big( y_i\ log(\sigma(z_i)) + (1-y_i)\ log(1-\sigma(z_i)) \big)$$

Implement the hypothesis function $\sigma(z_i) = \frac{1}{1+e^{-z_i}}$ where $z_i=x_iw^T$

In [232]:
def h(x, w):
    z_i = np.dot(x, w.T)
    sigma = 1 / (1 + np.exp(-z_i))
    return sigma

Implement the cost using hypothesis

In [234]:
def cost(x, y, w):
    j_w_1 = (1 - y)*np.log(1 - h(x, w))
    j_w_2 = y*np.log(h(x, w))
    j_w_3 = j_w_2 + j_w_1
    j_w = -1*np.average(j_w_3)
    return j_w

Partial derivative of $w_j$:

$$\frac{\partial J(w)}{\partial w_j} = \frac{1}{m} \sum_{i=1}^{m}(\sigma(z_i) - y_i) * x_i^j$$

The vectorized form to get all w's at once:

$$ \nabla J(w)=\frac{1}{m} X^T (\sigma(z) - y) $$
In [236]:
def grad(x, y, w):
    d_j_w_1 = 1/len(y)
    d_j_w_2 = np.dot(x.T, (h(x, w) - y))
    d_j_w = (d_j_w_1*d_j_w_2)
    return d_j_w

Implement the gradient descent algorithm:

In [238]:
def gd(x, y, x_t, y_t, w, alpha, epochs):
    J = {}    # loss function with training set
    J_t = {}  # loss function with test set
    for i in range(epochs):
        w_grad = grad(X_train, y_train, w).T  # compute the gradinet
        w = w - alpha*w_grad                  # update weights
        if i % 10 == 0:                       # update cost calculations every 10 iterations
            J[i] = cost(X_train, y_train, w)  # compute the cost for training
            J_t[i] = cost(X_test, y_test, w)  # compute the cost for testing
    return w, J, J_t

def accuracy(x, y, w):
    tp = sum(x[y == 1] == 1)
    tn = sum(x[y == 0] == 0)
    acc = (tp + tn)/len(x)
    return acc, tp + tn

5. Compute confusion matrix¶

  1. True positives (TP) – the number of objects with class = 1 that are classified as 1
  2. True negatives (TN) – the number of objects with class = 0 that are classified as 0
  3. False positives (FP) – the number of objects with class = 0 that are classified as 1
  4. False negatives (FN) – the number of objects with class = 1 that are classified as 0
        True Values
          0     1
        ------------
y_hat 0 | tn    fn |
      1 | fp    tp |
        ------------

Return a 2d array as follows:

array([[tn, fn], [fp, tp]])

In [248]:
def confusion_matrix(y, y_hat):
    tp = sum(y[y_hat == 1] == 1)
    fp = sum(y[y_hat == 1] == 0)
    fn = sum(y[y_hat == 0] == 1)
    tn = sum(y[y_hat == 0] == 0)
    return [[tn, fn], [fp, tp]]

6. Conclusions¶

  • The normalization of the data was interesting, the one applied resulted in decent results, neverthless im curious about other types like standarization or robust standarization to diminish the outlier effect (instead of mean, median, instead of standar deviation, interquantile range). I never gave a serious thought to image processing but i guess it could be good to try more variations of that.

  • The threshold for classification (in the example 0.5) is a standard decision i guess, i didnt conducted more tests with variations of it, but i did thought on implementing 10 different variations and plot the ROC curve and calculate the AUC, but ran out of coffee and when i came back i realized i had a lot of work to do with the TOG so, maybe later.

  • With 1 and 7 digits the model achieve a decent accuracy with 2,000 iterations, and in the loss plot the values for the test set converges like in the train which is an indicative of low overfitting effects for that particular experiment, although will be better to have another set with data never seen by the model, or crossvalidation techniques or add random perturbations/rotations to the existing set of images.

  • I conducted a 2nd quick test, with the digits 3 and 8 , which are kind of similar to each other, like the 1 and 7. Did it with the same normalization, alpha and epochs. The results where less good but still not too bad for a very simple model, that two classes (3, 8) had less occurrences so it could be an imbalace problem therefore some weight on the classes could be useful, also, it could be a sign that alpha and epochs were "better" for 1 and 7 example case and not to other. Other interesting observation is that loss convergence in test set was more convex, but at the end showed a little evidence of increasing values in direction to divergence, this could be due to "easier" examples in the test set and therefore a sign that it could be useful to have a K-fold crossvalidation.
